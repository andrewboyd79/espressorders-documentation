EspressOrders

A production-style coffee ordering platform — built to demonstrate cloud engineering, microservices architecture, and DevOps automation.

Context

The monolith started life as my final project for a Diploma in Full Stack Web Development at the Code Institute — a standard Django application, functional but built to meet a curriculum brief rather than for production. Rather than build a clean demo from scratch, I chose to work with this existing, imperfect codebase. That is the same decision a DevOps or cloud engineer faces on day one at any real company.

The goal was to practise and improve automation, infrastructure, and cloud skills on something real, messy, and at times resistant to the work. Every constraint the codebase imposed made the decisions more interesting and the solutions more representative of real engineering.

"I could have used a demo project. I chose not to — because the interesting problems only appear when the codebase isn't designed for you."

Key Decisions

Senior thinking, not just senior building

Every major architectural choice is documented as an Architecture Decision Record — the reasoning, the alternatives considered, and the trade-offs accepted.

Decomposition

Strangler Fig over big-bang rewrite

Incremental extraction with the monolith running in parallel — zero downtime, reversible at every step.

CI/CD

CodeBuild over CodePipeline

Consistent with the existing ecosystem, $2/month cheaper, no functional gap for this use case.

Mobile

Expo preview profile over App Store

iOS simulator build via EAS — no Apple Developer account required, full CI/CD pipeline demonstrated.

Built on a repeatable platform

Every service follows the same repo structure, the same CloudFormation pattern, the same CI/CD approach. The entire environment spins up or down with a single command — the same pattern a platform team would put behind a button in an internal developer portal.
